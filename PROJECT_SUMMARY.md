## 一、架构演进

### Day 1: GPT-2 → BART 重构

**问题发现**
- 租用 RTX 4090 服务器跑 GPT-2 模型，效果反而变差
- 排查发现：训练数据全为繁体字、唐诗宋词混在一起、五言七言混在一起

**解决**
- 放弃 GPT-2，改用 BART Encoder-Decoder 架构
- 原因：GPT-2 是单向续写，关键词作为 Prompt 被强制拼接为诗句开头，导致两个固有问题：
  1. 关键词永远出现在首句第一个字
  2. 首行缺字（2字关键词 + 模型续写2字 = 4字，五言需5字）
- BART 的 Encoder-Decoder 天然分离输入输出

### Day 2: 数据扩展 + 严格格式分类

**数据清洗**
- 繁体→简体：1,155,403 处转换
- 注释清洗：移除古籍引用、异文注释、补字符号
- 格式严格分类：0.95 阈值，95%以上行必须统一字数

**数据量**
- 唐诗 56,018 首（清洗后）
- 严格五言 28,503 首，严格七言 21,218 首，宋词 20,583 首

**训练策略**
- 三种训练模式：场景描述→全诗、标题→全诗、首联→全诗
- 指令格式训练："写诗：" / "写一首词，" 前缀防止 Prompt 泄露
- 三个独立模型：七言(63,245对)、五言(85,060对)、宋词(41,070对)

### Day 3: 前端适配 + 格式严格化

**同事前端合并**
- 合并古风前端：朱红主色调(#C43D3D)、印章标题、纸纹背景
- 新增诗词库 API（3000首），支持按作者/词牌筛选

**严格格律控制**
- 唐诗：字数或句数不达标→无限重试直到合格
- 宋词：至少 N 句，不限字数上限
- 唐诗句数：绝句(4句)、律诗(8句)、排律(12句)
- 唐诗/宋词独立句数控制

## 四、核心技术方案

### 1. 指令格式训练
```
输入: "写诗：描写春风的意境。"
输出: 诗句（不含"春风"原文）
```
`写诗：` 前缀明确告知模型"这是创作指令"，防止输入泄露到输出。

### 2. 首联扩写
```
用户输入: "春风" (2字)
  ↓ 检索索引
找到: "南陌春风早，东邻旭日新。" (真实唐诗首联)
  ↓ BART续写
输出: 完整七言诗
```
8,086 个关键词→真实诗歌首联索引。

### 3. 无限重试
```
生成 → 检查字数(7字/行) → 检查句数(4/8/12) → 不合格→丢弃重新生成
```
确保输出 100% 符合格律。

### 4. 严格格式分类
```
阈值从 0.8 → 0.95
95%以上行必须统一字数才收入训练集
```
