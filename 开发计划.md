# 诗词自动生成系统 - 开发计划

## Context

基于 `项目要求.md` 的三大功能需求，搭建一个完整的 Web 应用：
- **F1**：关键字 → 生成符合韵律的诗词
- **F2**：场景描述 → 文本/情感分析 → 生成诗词
- **F3**：上传图片 → 图像识别 → 生成诗词

技术选型已确认：**Flask + PyTorch + 预训练模型微调（GPT2-Chinese）**。

## 3. 模型选型

| 用途 | 模型 | 说明 |
|------|------|------|
| 诗词生成（主力） | `uer/gpt2-chinese-poem` | 在 80 万首古诗上预训练，180MB，HuggingFace 直接加载 |
| 诗词生成（备选） | LSTM (自训练) | 满足项目要求中的 LSTM 技术指标，50MB，CPU 可用 |
| 图像识别 | `OFA-Sys/chinese-clip-vit-base-patch16` | 中文 CLIP，图片→中文描述 |
| 关键词提取 | Jieba TF-IDF/TextRank | 经典中文分词 |
| 情感分析 | SnowNLP | 轻量级中文情感分析 |
| 韵律校验 | pypinyin | 拼音转换 + 韵脚比对 |

## 4. API 设计

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/health` | 健康检查 |
| POST | `/api/generate/keyword` | F1：关键字→诗词 |
| POST | `/api/generate/scene` | F2：场景→诗词 |
| POST | `/api/generate/image` | F3：图片→诗词 |

统一返回格式：`{"success": bool, "poem": [...], "rhyme_score": float, ...}`

## 5. 开发阶段（5 个 Sprint）

### Sprint 1：环境 + 数据 + LSTM 基线
- 创建虚拟环境，安装 PyTorch + Flask
- 下载唐诗数据集（chinese-poetry 仓库，约 5.8 万首）
- 编写数据预处理脚本
- 训练 LSTM 基线模型
- 搭建 Flask 骨架 + `/api/generate/keyword` 端点
- **可演示**：命令行生成诗词

### Sprint 2：GPT-2 微调 + 韵律校验
- 加载 `uer/gpt2-chinese-poem`
- 微调训练 GPT-2
- 实现模型切换（GPT-2 / LSTM）
- 集成 pypinyin 韵律校验
- **可演示**：GPT-2 高质量诗词生成

### Sprint 3：NLP 管道 + 场景生成 (F2)
- Jieba 关键词提取
- SnowNLP 情感分析
- 场景→分析→诗词完整链路
- `/api/generate/scene` 端点
- 前端场景输入 Tab
- **可演示**：输入场景描述，输出诗词

### Sprint 4：图像识别 + 图片生成 (F3)
- 加载 Chinese-CLIP
- 图片→中文标签→诗词链路
- `/api/generate/image` 端点
- 前端图片上传 Tab（拖拽上传）
- **可演示**：上传图片，输出诗词

### Sprint 5：前端美化 + 测试 + 文档
- Bootstrap 5 响应式布局
- 加载动画、复制按钮
- 单元测试
- README 完善
- 性能优化（模型预热、缓存）

## 7. 风险与应对

| 风险 | 应对 |
|------|------|
| CPU 跑不动 GPT-2 | LSTM 备选方案，模型量化 |
| HuggingFace 下载失败 | 设置 `HF_ENDPOINT=https://hf-mirror.com`（国内镜像） |
| Chinese-CLIP 太大 | ResNet-18 + 中文标签映射表作为降级方案 |
| 生成质量差 | 降低 temperature，生成多个候选后选最优 |

## 验证方式

1. `python run.py` → 浏览器打开 `http://localhost:5000`
2. 关键字 Tab：输入"春" → 点击生成 → 验证返回四句诗且押韵
3. 场景 Tab：输入"月下独饮伤感" → 验证提取关键词 + 情感分析正确
4. 图片 Tab：上传风景图 → 验证识别标签与图片相关
5. `pytest tests/` → 所有测试通过
