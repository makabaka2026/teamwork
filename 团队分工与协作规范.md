# 诗词自动生成系统 — 团队分工与协作规范

---

## 一、项目简介

基于深度学习的唐诗宋词自动生成 Web 应用，三大功能：

| 编号 | 功能 | 输入 | 技术链路 |
|------|------|------|----------|
| F1 | 关键字生成 | 一个中文关键词 | GPT-2 / LSTM → 诗词 |
| F2 | 场景生成 | 一段场景描述 | Jieba 分词 + SnowNLP 情感分析 → GPT-2 → 诗词 |
| F3 | 图片生成 | 上传一张图片 | Chinese-CLIP 识别 → GPT-2 → 诗词 |

**仓库地址：** `https://github.com/makabaka2026/teamwork.git`  
**技术栈：** Flask + PyTorch + HuggingFace Transformers + Bootstrap 5

## 三、角色 A — 后端引擎

### 你要做什么

1. **诗词生成核心** (`poetry_generator.py`)
   - GPT-2 模型加载、推理、参数调优（temperature/top_p/top_k/repetition_penalty）
   - LSTM 模型加载、逐字生成完善
   - 模型自动降级：GPT-2 失败 → LSTM → 回退诗词库
   - 扩充回退诗词库（当前只有5个关键字）

2. **LSTM 网络** (`lstm_model.py`)
   - 三层 LSTM + Embedding + FC
   - 完善 `generate()` 方法（当前是简化版）

3. **韵律校验** (`rhyme_checker.py`)
   - pypinyin 押韵评分（0-1分）
   - 平仄检查
   - 区分唐诗（偶数句押韵）和宋词（词牌规则）

4. **文本格式化** (`poetry_formatter.py`)
   - 生成文本清洗、分行、标点规范
   - 竖排显示转换

### 你的接口（给角色 D 调用）

```python
# 全部在 backend/services/ 和 backend/utils/ 下

PoetryGenerator()
    .load_model(model_type="gpt2")           # → None, 失败自动降级
    .generate(prompt, num_lines, temperature) # → str 原始文本
    .model_type                               # → "gpt2" | "lstm" | "none"

TextAnalyzer()
    .extract_keywords(text, top_n=5)          # → list[str] 关键词

SentimentAnalyzer()
    .analyze(text)                            # → {label, score, mood}

ImageRecognizer()
    .load_model()                             # → None
    .recognize(image_path)                    # → (list[str], str) 标签+描述

RhymeChecker()
    .score(poem_lines)                        # → float 0.0-1.0

PoetryFormatter()
    .format(raw_text, num_lines)              # → list[str] 诗词行
```

## 四、角色 B — 数据与模型训练

### 你要做什么

1. **数据预处理** (`data_preprocessing.py`)
   - 从 chinese-poetry 仓库下载唐诗宋词 JSON 数据
   - 构建字符级词汇表（8000字）
   - 生成 LSTM 训练格式（`.npz`）和 GPT-2 训练格式（`.jsonl`）

2. **LSTM 训练** (`train_lstm.py`)
   - 训练 3 层 LSTM，50 epochs
   - 产出权重文件 `lstm_tang.pth` → 放到 `backend/checkpoints/lstm/`

3. **GPT-2 微调** (`train_gpt2.py`)
   - 基于 `uer/gpt2-chinese-poem` 微调，3 epochs
   - 产出模型 `gpt2-poem/final/` → 放到 `backend/checkpoints/gpt2-poem/`

4. **模型评估** (`evaluate.py`)
   - 计算 Perplexity 和 BLEU 分数
   - 输出生成样本供人工评估
   - 将评估数据写到报告里

### 数据流向

```
chinese-poetry 仓库 (5.8万首唐诗 + 2.1万首宋词)
    │  data_preprocessing.py
    ├── tang_lstm.npz + vocab.pkl   →  train_lstm.py  →  lstm_tang.pth
    └── tang_gpt2.jsonl             →  train_gpt2.py  →  gpt2-poem/final/
                                                              │
                                ┌─────────────────────────────┘
                                ▼
                      backend/checkpoints/  (角色 A 从这里加载)
```

### 你的环境

```bash
python -m venv D:\teamwork-env-training\.venv
source D:\teamwork-env-training\.venv\Scripts\activate
pip install -r requirements-training.txt --extra-index-url https://download.pytorch.org/whl/cpu
export HF_ENDPOINT=https://hf-mirror.com

# 下载数据集
cd training
git clone --depth 1 https://github.com/chinese-poetry/chinese-poetry.git data/raw/chinese-poetry
# 将 data/raw/chinese-poetry/json/ 下的 tang 和 ci 目录移到 data/raw/

# 预处理
python data_preprocessing.py --format both

# 训练
python train_lstm.py --epochs 50
python train_gpt2.py --epochs 3

# 评估
python evaluate.py
```

依赖清单（`requirements-training.txt`）：torch, transformers, datasets, pandas, numpy, sacrebleu, peft, accelerate, tqdm

## 五、角色 C — 前端

### 你要做什么

1. **三个 Tab 页面** (`index.html`)
   - 关键字 Tab：输入框 + 行数/风格/创造力参数
   - 场景 Tab：文本域 + 参数 + 分析结果展示区
   - 图片 Tab：拖拽上传区 + 图片预览 + 参数

2. **交互逻辑** (`main.js`)
   - F1 关键字生成：收集参数 → POST JSON → 渲染诗词
   - F2 场景生成：收集参数 → POST JSON → 渲染诗词+分析结果
   - F3 图片生成：FormData 上传 → 渲染诗词+识别标签
   - 复制按钮、重新生成按钮
   - 加载动画、错误提示

3. **样式** (`style.css`)
   - 诗词展示区：竖排显示、逐字动画
   - 图片上传区：拖拽高亮、预览缩略图
   - 移动端响应式适配

### 你对接的 API（角色 D 提供）

| 方法 | 路径 | 你需要传什么 | 你会收到什么 |
|------|------|-------------|-------------|
| GET | `/api/health` | 无 | `{status, model, gpu_available}` |
| POST | `/api/generate/keyword` | `{keyword, num_lines?, style?, temperature?}` | `{success, poem[], rhyme_score, ...}` |
| POST | `/api/generate/scene` | `{scene_text, num_lines?, temperature?}` | `{success, poem[], extracted_keywords, sentiment, ...}` |
| POST | `/api/generate/image` | FormData: image + num_lines + temperature | `{success, poem[], recognized_labels, ...}` |

**详细请求/响应格式见第七章 API 接口规范。**

## 七、API 接口规范（角色 C ↔ 角色 D 的契约）

**以下字段名、类型、约束，双方必须严格遵守。修改需同步通知对方。**

### 通用规则

```
统一成功: {"success": true, ...}
统一失败: {"success": false, "error": "原因"}
HTTP 状态码: 200(成功) / 400(参数错误) / 500(服务端错误) / 503(模型未就绪)
```

### F1 关键字生成

```
POST /api/generate/keyword
Content-Type: application/json
```

| 字段 | 类型 | 必填 | 约束 | 默认 |
|------|------|------|------|------|
| keyword | string | 是 | 1-10个汉字 | — |
| num_lines | int | 否 | 4 或 8 | 4 |
| style | string | 否 | "tang" 或 "song" | "tang" |
| temperature | float | 否 | 0.1-1.5 | 0.8 |
| model_type | string | 否 | "gpt2" 或 "lstm" | "gpt2" |

成功响应：
```json
{
    "success": true,
    "poem": ["春风得意马蹄疾，", "一日看尽长安花。"],
    "keyword": "春",
    "rhyme_score": 0.92,
    "model": "gpt2",
    "generation_time_ms": 1234
}
```

### F2 场景生成

```
POST /api/generate/scene
Content-Type: application/json
```

| 字段 | 类型 | 必填 | 约束 | 默认 |
|------|------|------|------|------|
| scene_text | string | 是 | 1-500字 | — |
| num_lines | int | 否 | 4 或 8 | 4 |
| temperature | float | 否 | 0.1-1.5 | 0.8 |

成功响应：
```json
{
    "success": true,
    "poem": ["江畔柳如烟，", "春风拂面暖。"],
    "extracted_keywords": ["春天", "江边", "柳树"],
    "sentiment": {"label": "positive", "score": 0.87, "mood": "明快豪放"},
    "rhyme_score": 0.88,
    "model": "gpt2",
    "generation_time_ms": 2100
}
```

### F3 图片生成

```
POST /api/generate/image
Content-Type: multipart/form-data
```

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| image | file | 是 | jpg/png/gif/webp/bmp，≤10MB |
| num_lines | int | 否 | 4 或 8 |
| temperature | float | 否 | 0.1-1.5 |

成功响应：
```json
{
    "success": true,
    "poem": ["孤舟蓑笠翁，", "独钓寒江雪。"],
    "image_description": "山水风景、孤舟独钓、雪景寒梅",
    "recognized_labels": ["山水风景", "孤舟独钓", "雪景寒梅", "田园乡村", "竹林深处"],
    "rhyme_score": 0.85,
    "model": "gpt2",
    "generation_time_ms": 2800
}
```

## 十、Git 协作规范

```
main                    ← 最终稳定版本，只有 D 合并
  ├── sprint-1           ← 当前基线
  ├── backend-engine     ← A 开发分支
  ├── model-training     ← B 开发分支
  ├── frontend           ← C 开发分支
  └── api-tests          ← D 开发分支
```

**规则：**
1. 每人每天至少 push 一次自己的分支
2. 合并到 main 前在 PR 里 @ 另外三人 review
3. commit message 格式：`[模块] 简短描述`，例如 `[backend] 完善 GPT-2 生成逻辑`
4. API 接口字段有修改时，在群里同步 @C（前端）
